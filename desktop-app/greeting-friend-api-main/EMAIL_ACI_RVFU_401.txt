Il flusso OAuth2 viene completato correttamente:
• POST /sso/json/authenticate → tokenId ricevuto
• POST /sso/oauth2/authorize → authorization code ricevuto
• POST /sso/oauth2/access_token → id_token e access_token ricevuti (entrambi validi)

Abbiamo effettuato test sia con id_token sia con access_token come Bearer, ottenendo lo stesso esito (401).
Inoltre, il server Apache cancella i cookie di sessione (Max-Age=0) nella risposta.
